提高设计产品能力的一些观点

发表于 2017-01-05 | 分类于 随笔 | 阅读次数

刚刚逛知乎，看到一个人说 UI 如何提升自己的问题，我想这个东西放到其他地方也是合适的。比如产品设计、程序设计等等。

可能楼主自己在做UI设计，尤其是在构思和落地的时候想的比较少。平时也对这个行业的关注度不够的原因造成的。在如今的互联网设计中，大部分的设计师是称不上“大神级”UI设计师，很多在行业里做的不错的设计师，其实也是通过自身努力和对行业的高关注度，做过一些大家耳熟能详的产品而被人熟知的。我个人的观点是这批人最多是“网红设计师”，还没有达到能影响整个行业的实力。

楼主也坦露心声的说出了很多设计师在做设计的思维方式，各种抄。今天运气好点开站酷、花瓣能找到个与需求相似度很高的界面就抄抄抄，这种方式很短时间高效的解决现阶段的问题，但是长此以往是会暴露出缺乏原创能力、设计基础差、思维方式迟钝等负面问题的。

个人建议楼主在以后的设计中，尝试比较古老的笨办法。先动脑不接触任何外在元素的情况把自己脑子所想到的设计方向都写下来，在根据自己写的内容进行头脑风暴，产出一些关键词、核心元素等内容。然后根据这些筛查出来的内容有针对性的去结合现在互联网比较热门的应用进行学习研究，分析问题和解决方法。最后在绘制草图，转到电脑上进行精准的界面设计。再初稿形成后，开会进行评审和提意见，修整方案。把每一次的评审记录备案留存下来。确定整体产品设计，以每次评审出现的问题和调整的思路进行总结复盘。最终完成整个项目~
